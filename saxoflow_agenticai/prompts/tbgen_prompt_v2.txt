=== TESTBENCH GENERATION PROCESS ===

1. PARSE THE DUT
   - List all input ports, output ports, their widths
   - Identify clock and reset (polarity, sync/async)
   - Identify protocol (APB handshake, counters, FSM, etc.)

2. MODULE STRUCTURE
   - Single testbench module (name: tb, or dut_tb)
   - Declare DUT inputs as reg; outputs as wire
   - Use integer for loop counters and test tracking only
   - NO: parameter declarations, SystemVerilog logic/bit/typedef, automatic/begin-end blocks
   - YES: reg, wire, integer types only

3. CLOCK GENERATION (if needed)
   - `always #(PERIOD/2) clk = ~clk;` (NOT forever)
   - Initialize clk to 0 in initial block
   - Choose PERIOD so simulation runs in reasonable time

4. RESET HANDLING (if needed)
   - Drive presetn/reset_n in initial block
   - Match spec polarity (active-low usually means start at 0, pulse for N cycles, then 1)
   - Example: presetn = 0; #(10*PERIOD) presetn = 1; to hold reset for 10 clocks
   - Ensure reset releases before stimulus starts

5. DUT INSTANTIATION
   - Use named port connections: .clk(clk), .presetn(presetn), ...
   - Check exact port names and widths from RTL
   - If DUT has no parameters, do not use #() suffix

6. VCD DUMP (mandatory)
   - Add in initial block EARLY:
     $dumpfile("tb.vcd");
     $dumpvars(0, tb);
   - Without this, no waveform output

7. STIMULUS DESIGN (comprehensive directed tests)
   - Initialize all inputs to spec-defined defaults (usually 0)
   - Create test phases with clear comments
   - Phase 1: Reset, default behavior (let DUT idle with valid defaults)
   - Phase 2: Basic functionality (single transaction, read/write, counter up/down)
   - Phase 3: Edge cases: boundaries (0, max, wrap), protocol corners (psel without penable, back-to-back transfers)
   - Phase 4: Stress (long sequences, repeated operations, mode switching)
   - Use bounded for-loops (explicit iteration counts, not infinite)
   - Use $display to print test progress and intermediate results

8. OUTPUT CHECKING (mandatory)
   - Read outputs into temp wires/integers
   - Compare against expected values
   - $display("TEST: clock cycle %d, output=%b, expected=%b", cycle, out, expected);
   - For failures: $display("ERROR: ...");
   - Keep checks simple and deterministic; avoid randomness unless bounded and seeded
   - NO: assertions, $strobe, UVM
   - YES: $display, $monitor (if monitoring specific signals)

9. PROTOCOL VERIFICATION
   - If handshake (e.g., APB): verify pready timing, valid transfer windows, pslverr setups
   - If counter: verify increment, decrement, overflow, saturation per spec
   - If interrupt: verify assertion on condition, clear semantics
   - Include at least 5-10 directed test cases

10. SIMULATION END
    - After all tests, print summary: $display("TEST COMPLETE");
    - Call $finish; (NOT $stop)

11. STYLE
    - Single testbench module, self-contained
    - Clear test comments: // TEST: <name>
    - No DPI, PLI, vendor pragmas
    - Portable; runs on Icarus Verilog and Verilator

12. IF FEEDBACK PROVIDED
    - Fix all port/width mismatches
    - Add missing $dumpfile/$dumpvars if absent
    - Improve reset pulsing if weak
    - Expand stimulus if insufficient coverage
    - Simplify or fix clocking if issues found

=== OUTPUT FORMAT ===
Output ONLY the complete single testbench module (module...endmodule). NO explanations, Markdown, or code fences.
