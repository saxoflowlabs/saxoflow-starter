=== STEP-BY-STEP GENERATION PROCESS ===

1. ANALYZE THE SPEC
   - Identify all inputs, outputs, buswidth, behavior
   - Note reset type (sync/async, polarity, use if necessary)
   - Identify all timing relationships and state transitions
   - Call out arithmetic (e.g., counter overflow)
   - Confirm any protocol (APB, AXI, custom handshake)

2. DESIGN STRUCTURE
   - Single module, no hierarchy
   - Module name matches spec
   - Explicit port declarations (input/output, widths, directions)
   - Combinational logic ONLY in `always @*` with nonblocking-free assignments (=)
   - Sequential logic ONLY in `always @(posedge clk)` with nonblocking (<=)
   - One driver per reg; never read-modify-write in comb
   - Default assignments in `always @*` to avoid latches
   - `case` includes `default`

3. CLOCK AND RESET
   - Use single clock if possible; no clock gating, use enables
   - Reset only if spec requires; match polarity and sync/async
   - Reset action: set all regs to spec-defined defaults (0, 1, or calculated values)
   - Synchronous reset: simple if inside `always @(posedge clk)`
   - Async reset: use `always @(posedge clk or negedge reset)` with synchronous logic in reset branch

4. WIDTHS AND ARITHMETIC
   - Every port, wire, reg has explicit width [MSB:LSB]
   - Arithmetic growth: adder output = max(A,B) + 1; counter overflow at bit width
   - Truncation only when intentional and documented
   - No signed/unsigned confusion; pick one consistently

5. INTERFACES
   - All handshake signals match spec exactly (names, polarity, timings)
   - All register bits map to spec register map
   - All outputs combinational or registered per spec

6. SYNTHESIS SAFETY
   - NO: initial, #delay, $display, forever, fork/join, tri-state, force/release, specify, `define without `ifndef SYNTHESIS
   - YES: parameters, functions (helpers), descriptive comments
   - Portable: avoid $clog2; use helper function from guidelines if needed

7. CODING STYLE
   - Port order: clk, reset (if any), inputs, outputs
   - Signal naming: clear, spec-aligned, _n for active-low
   - Comments: inline for key logic/IO; no redundant comments
   - No Verilog keywords as signal names (reset, clock, enable, etc. as prefixes)

8. IF FEEDBACK IS PROVIDED
   - Address every bullet point
   - Fix all syntax, width, reset, latch, FSM, tool-specific issues
   - Preserve correct intent unless spec/review says otherwise

=== OUTPUT FORMAT ===
Output ONLY the complete RTL module (module...endmodule). NO explanations, Markdown, or code fences.
