=== REPORT STRUCTURE ===

Create these sections with exact headings:

1. Specification & Design Goals
   - Brief summary of design intent and scope from spec

2. RTL Generation Phase
   - Inputs: spec
   - Actions: RTL generated
   - Issues from review: 3-5 bullets
   - Status: pass/fail

3. RTL Review Results  
   - Syntax/language compliance?

   - Latches avoided?
   - Tool compatibility (Yosys, OpenROAD)?
   - Recommendations: 2-3 bullets

4. Testbench Generation Phase
   - Coverage: tests included (reset, basic, edge cases, stress)?
   - Clock/reset generation adequate?
   - VCD dump configured?
   - Issues from review: 3-5 bullets

5. Testbench Review Results
   - Structure/module layout correct?
   - Stimulus coverage sufficient?
   - Output checking adequate?
   - Portability (Icarus, Verilator)?

6. Formal Properties Phase
   - Coverage: properties for reset, logic, boundaries, outputs?
   - Quality: specific vs trivial?
   - Issues from review: 2-4 bullets

7. Simulation & Debug Results
   - Overall pass/fail status
   - Quote short log line if failed
   - VCD produced: yes/no
   - Suggested fixes (map to RTLGenAgent, TBGenAgent, UserAction)

8. Project Status & Next Steps
   - Ready for synthesis: yes/no
   - Outstanding issues (prioritized): bullet list
   - Recommended next agent/action for each issue

=== OUTPUT STYLE ===
